## Практика. Реализация API согласно Swagger

Реализовать CRUD операции для вопросов для Quiz game, в соответствии с super admin api и логику игры Quiz game (см. ниже).
Подробнее про quiz game смотрите в 23 - Online Quiz-game.​
Время жизни access токена должно быть не менее 5 мин. Ip-restriction нужно отключить.

Quiz game:
Реализовать логику создания игры, подключения к игре, ответов на вопросы, завершения игры и т.д. , в соответствии с  public api.

Пояснения:
1. Я как зарегестрированный пользователь могу соревноваться в квизе попарно (с другим зарегестрированным пользователем);
2. Я нажимают кнопку: соревноваться (join);
3. Если есть игрок в ожидании - создаётся пара: я + этот игрок;
4. Если нет, я становлюсь игроком в ожидании и могу стать парой для следующего, кто нажмёт соревноваться;
5. Эндпоинт GET /pair-game-quiz/pairs/my-current возвращает активную игру текущего пользователя (того, кто делает запрос)  в статусе "PendingSecondPlayer" или "Active".
Эндпоинт GET /pair-game-quiz/pairs/:id возвращает игру текущего пользователя (того, кто делает запрос)  в любом статусе.
Если игра в статусе ожидания второго игрока (status: "PendingSecondPlayer") - поля secondPlayerProgress: null, questions: null, startGameDate: null, finishGameDate: null;
6. Когда фиксируется пара, система рандомно выбирает из списка 5 вопросов и фиксирует их для пары  (одни и теже 5 вопросов для каждого игрока в паре).
Если пары еще нет ( поле status: "PendingSecondPlayer") - список вопросов не возвращаем (поле questions: null). Если к игре подлючился второй игрок (status: "Active") - возвращаем все 5 вопросов (поле questions: [...]).
Участники последовательно отвечают на вопросы. Одна попытка. Ответ либо правильный либо нет;
7. Победитель определяется по очкам:
За каждый правильный ответ по 1 баллу.
Допольнительный 1 балл, если ответил на все вопросы быстрее другого и хотя бы 1 вопрос отвечен правильно.
Дополнительный бал начислается в момент, когда игра завершается (последний игрок дает ответ на последний вопрос);
8. Пример 1:
Игрок А ответил быстрее Игрока Б:
- Игрок А: ✅🟥🟥✅🟥: 1 - 0 - 0 - 1 - 0  +1 (бонус, так как ответил быстрее играка Б) = 3;
- Игрок Б: ✅✅🟥✅🟥 1 - 1 - 0 - 1 - 0   +0 (нет бонуса) = 3;

- Игрок Б: 🟥✅🟥🟥🟥0 - 1- 0 - 0 - 0 - +0 (нет бонуса, так как не быстрее игрока A ответил) = 1;
